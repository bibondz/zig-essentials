# zig-essentials v0.1.0 — first stable releases

Stable, explicit, minimal Zig libraries that should be in `std` but aren't.

Pinned to **LTS-0.16** (Zig 0.16.0). API surfaces are frozen at this version — no breaking changes until 1.0.0.

---

## 📦 What's in this release

Three libraries ship in this release, each with a frozen public API:

### [`uuid`](https://github.com/bibondz/zig-essentials/tree/main/uuid) — RFC 4122/9562 v4 + v7

Stable, minimal, explicit UUID library.

```zig
const std = @import("std");
const uuid = @import("uuid");

var prng = std.Random.DefaultPrng.init(0xdeadbeef);
const id = uuid.Uuid.v4(prng.random());
// e.g. "f4d1c2b3-a4b5-4687-9abc-def012345678"

const id_v7 = uuid.Uuid.v7At(prng.random(), 1718000000000);
// time-ordered random, sortable

_ = id.version();      // .v4
_ = id.isRfc4122();    // true
```

**23/23 tests pass.** See [uuid/README.md](https://github.com/bibondz/zig-essentials/blob/main/uuid/README.md).

### [`cli`](https://github.com/bibondz/zig-essentials/tree/main/cli) — struct-based argument parser

Declarative schema, parses into your struct. Fills the gap left by `std.process.Args` (which only has a string splitter).

```zig
const std = @import("std");
const cli = @import("cli");

const Args = struct {
    verbose: bool = false,
    output: []const u8 = "",
    file: []const u8 = "",
};

const args: Args = try cli.parse(Args, .{
    .name = "mytool",
    .version = "1.0.0",
    .opts = &.{
        .{ .name = "verbose", .short = 'v', .kind = .flag },
        .{ .name = "output", .short = 'o', .kind = .option },
    },
    .positionals = &.{
        .{ .name = "file" },
    },
}, std.os.argv);
```

Supports long/short flags, combined shorts, options via `=` or space, positionals (required/optional/many), auto `--help` / `--version`. **18/18 tests pass.** See [cli/README.md](https://github.com/bibondz/zig-essentials/blob/main/cli/README.md).

### [`toml`](https://github.com/bibondz/zig-essentials/tree/main/toml) — TOML 1.0 parser

Parse TOML into a dynamic `Value` tree (tagged union). Arena-backed. Supports all common TOML 1.0 features.

```zig
const std = @import("std");
const toml = @import("toml");

var arena = std.heap.ArenaAllocator.init(std.heap.page_allocator);
defer arena.deinit();

const v = try toml.parse(arena.allocator(),
    \\[package]
    \name = "myapp"
    \\[dependencies]
    \zig = "0.16.0"
);

const name = toml.getString(v.table, "name").?;        // "myapp"
const zig_dep = toml.getString(
    toml.getTable(v.table, "dependencies").?,
    "zig"
).?;  // "0.16.0"
```

Supports: strings (basic/literal/multi-line), integers, floats, booleans, datetimes (as RFC 3339 string), arrays, tables (regular/inline/nested/array-of), dotted keys, quoted keys, comments. **24/24 tests pass.** See [toml/README.md](https://github.com/bibondz/zig-essentials/blob/main/toml/README.md).

## 🚀 Install

Add as a dependency in your `build.zig.zon`:

```zon
.{
    .name = .myapp,
    .version = "0.1.0",
    .dependencies = .{
        .uuid = .{
            .url = "https://github.com/bibondz/zig-essentials/archive/refs/tags/uuid-v0.1.0.tar.gz",
            // hash will be filled by `zig build` on first use
        },
        .cli = .{
            .url = "https://github.com/bibondz/zig-essentials/archive/refs/tags/cli-v0.1.0.tar.gz",
        },
    },
    .paths = .{ ... },
}
```

*(Or copy `uuid/src/uuid.zig` directly — each lib is a single file + tests, no transitive deps.)*

## ✅ Quality

- **TDD:** 100% of features are covered by tests, written RED-first
- **CI stub:** `.forgejo/workflows/test.yml` (will run on Codeberg/Forgejo + GitHub Actions once enabled)
- **LTS pinned:** Zig 0.16.0 (the latest stable, not 0.17-dev)
- **No LLM-generated code:** human-authored only, aligned with Zig org policy

## 🛡️ Stability promise

- `Uuid` layout (`extern struct`, 16 bytes) is frozen
- `parse(T, schema, arena, argv)` signature is frozen
- New features = new functions, not signature changes
- Deprecation cycle: minimum 2 minor versions before removal
- See [PROJECT_PLAN.md §3](https://github.com/bibondz/zig-essentials/blob/main/PROJECT_PLAN.md) for full versioning policy

## 🚫 Out of scope (deferred to v0.2.0+)

| Lib | Not yet | v0.2.0 plan |
|---|---|---|
| `uuid` | v1, v3, v5, v6, v8; `urn:uuid:` prefix; database types | — (out of scope, RFC 4122/9562 only needs v4 + v7) |
| `cli` | subcommands; `--no-flag` negation; custom validators; type coercion | ✅ all four planned |
